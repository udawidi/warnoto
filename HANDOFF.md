**Vendor aktif terakhir:** Claude | **Update:** 2026-07-21 WIB (lanjutan — insiden data)

## Tujuan / benang merah
WARNOTO adalah aplikasi gudang PLN (React, Vite 4, Supabase). Fokus: penyempurnaan UI bertahap dan migrasi Non-SAP UPT Surabaya secara review-first, bukan redesign besar.

## Keputusan arsitektur
- `App.jsx` masih besar; split internal ditunda sampai user menyetujui.
- Supabase `tadxodrzoquugnsyejld`; perubahan skema harus diusulkan dulu. Jangan drop `wa_sync_status`.
- Tailwind v4 via PostCSS, preflight off. Deploy hanya `git push main`.
- Sidebar: desktop 260/76px, drawer mobile <=768px; toggle di footer sidebar (lebar = « pojok kanan bawah, compact = »» tengah).
- Alur bisnis review-first; jangan membuat aksi turunan atau auto-approve tanpa persetujuan.
- Routing agent: jika ragu memilih `worker` atau `senior`, prioritaskan `worker`; gunakan `senior` hanya setelah kompleksitas/risiko terbukti atau `worker` mengeskalasi karena scope melampaui levelnya.
- Pembaruan `HANDOFF.md` wajib meminta dan memperoleh persetujuan user terlebih dahulu; jangan memperbaruinya otomatis, termasuk setelah tahap signifikan atau akhir sesi.
- Pak War: Groq (`llama-3.3-70b-versatile`), key selalu di-`.trim()` (pernah 401 gara-gara spasi bawaan `vercel env pull`); fallback data lokal saat AI eksternal gagal. Gaya jawaban humanis-korporat, jawab yang ditanya saja, list = `- **Nama** [kode] — stok X unit · Lokasi: Y`, dirender kartu `.ai-richlist`.
- Tipografi: floor 12px di semua CSS + inline style (kecuali ScanPublicView 10.5px, halaman print). JANGAN menambah teks <12px.
- Banner KPI seragam via kelas bersama `.kpi-banner` di `src/index.css` (proporsi compact seperti `.dashboard-maturity`: min-height 104px, radius 14px, angka 22px/800, gradient navy). Dipakai: Approval, Kapasitas Gudang, Forecast Stok, TUG; OperationsHero (Alat Berat + ATTB) diselaraskan proporsinya di `src/styles/operations.css`.
- Foto satpam & foto alat berat: data URL terkompres (satpam max 400px/±120KB via `compressImage` dari `src/lib/supabaseSync.js`) disimpan inline di jsonb master — TANPA perubahan skema.
- **Dark mode — pendekatan & hasil (2026-07-17, DI-COMMIT & DI-PUSH `81a50b9`).** Palet JS statik `C` (src/theme.js) di-prop-drill ke semua komponen; CSS mentah (index.css+operations.css) 99% hex literal, cuma segelintir pakai `var(--color-*)` → repaint 100% tidak proporsional dalam satu pass tanpa browser automation. Pendekatan: `C`/`sty` dibuat theme-aware via shadow lokal (`const C=theme==="dark"?C_DARK:C_LIGHT` di PLNWarehouse, sebelum `sty=makeSty(isMobile,C)`) — otomatis cover semua pemakai `C.xxx`/`sty.xxx` tanpa ubah child manapun. Percobaan pertama (MVP: cuma chrome/sidebar/Pak War/tombol approval) gagal user-test ("kacau, banyak tulisan tak terbaca") — root cause: `.app-shell` (App.jsx:4305) tidak pernah set `color`, jadi hampir semua teks mewarisi hitam default browser, invisible begitu background gelap. Perbaikan final: (1) `color:C.text` di `.app-shell`; (2) 4 kartu inline `background:"#fff"/"white"`→`C.surface`; (3) ~35 kelas CSS (workspace/dashboard/capacity/TUG/forecast/operations) dapat dark override koheren (darken-bg+lighten-text sekaligus, bukan patchwork); (4) toggle jadi saklar switch fisik di header kanan atas (App.jsx ~L4549, `.theme-switch*` index.css:398), dropdown lama dihapus. SENGAJA TIDAK disentuh: `.kpi-banner`+sidebar (sudah navy gelap), badge/alert saturasi kecil (self-contained, aman di kedua tema), halaman print (ScanPublicView/docBuilders — selalu light). Toggle persist `localStorage.warnoto_theme`, default light, tanpa auto-deteksi OS. User sudah verifikasi visual di browser → OK.
- **Bug sidebar SUPERADMIN cuma 2 menu (2026-07-17, DI-COMMIT & DI-PUSH `81a50b9`, sama dengan commit dark mode di atas).** Ditemukan setelah fajar dipromosikan ke SUPERADMIN: `isPengadaan`/`isUltgRole` (App.jsx:4256/4258) pakai `hasRole()` yang SENGAJA bypass SUPERADMIN untuk gate izin — tapi di sini dipakai untuk SELEKSI cabang sidebar, sehingga SUPERADMIN ikut kejaring cabang PENGADAAN (2 menu saja). Fix: ganti ke pengecekan role langsung (`currentUser?.role==="PENGADAAN"` / `ULTG_ROLES.includes(currentUser?.role)`), tidak menyentuh `hasRole()` di gate izin lain (itu masih benar sengaja bypass SUPERADMIN).
